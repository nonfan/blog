## Frontmatter

每篇文章开头需要使用 YAML 格式的 frontmatter 来定义元数据：

```yaml [文章配置示例]
---
title: 文章标题
description: 文章摘要描述
tags:
  - 标签1
  - 标签2
logo: https://example.com/logo.png
pinned: true
---
```

支持的字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 否 | 文章标题，默认取一级标题 |
| description | string | 否 | 文章摘要，默认取一级标题下的内容 |
| tags | string[] | 否 | 标签列表 |
| logo | string | 否 | 文章图标 URL |
| pinned | boolean \| number | 否 | 置顶设置，见下方说明 |
| type | string | 否 | 文章类型：`tech`（默认）或 `article` |

### 文章类型

`type` 字段用于区分不同类型的文章：

- `type: tech` - 技术文章（默认），显示目录、标签、导出 PDF 按钮
- `type: article` - 散文/随笔，隐藏目录和标签，更简洁的阅读体验

```yaml [散文类型示例]
---
title: 我的随笔
type: article
tags:
  - 随笔
---
```

### 置顶设置

`pinned` 字段支持两种类型：

- `pinned: true` - 置顶文章，但不指定顺序，按日期排序
- `pinned: 1` / `pinned: 2` / ... - 置顶文章并指定顺序，数字越小越靠前

排序优先级：数字置顶 > true 置顶 > 普通文章

```yaml [置顶示例]
# 第一个置顶位置
pinned: 1

# 第二个置顶位置
pinned: 2

# 置顶但不指定顺序
pinned: true

# 不置顶（默认）
pinned: false
```

## 标题

标题使用 `#` 符号，支持 1-6 级标题：

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

## 文本样式

**粗体文本** 使用 `**文本**` 或 `__文本__`

*斜体文本* 使用 `*文本*` 或 `_文本_`

***粗斜体*** 使用 `***文本***`

~~删除线~~ 使用 `~~文本~~`

这是一段普通文本，用于展示博客的排版效果。好的排版能让读者更专注于内容本身。段落之间保持适当的间距，行高设置合理，让阅读更加舒适。

```python [fibonacci.py]
def fibonacci(n: int) -> list[int]:
    if n <= 0:
        return []
    elif n == 1:
        return [0]
    
    fib = [0, 1]
    for i in range(2, n):
        fib.append(fib[i-1] + fib[i-2])
    return fib

print(fibonacci(10))
```

## 自定义容器

支持 VitePress 风格的自定义容器语法，使用 `:::type` 格式：

```markdown
::: info
信息内容
:::

::: tip
提示内容
:::

::: warning
警告内容
:::

::: danger
危险内容
:::

::: details 标题
可折叠的详情内容
:::
```

### 容器类型

| 类型 | 默认标题 | 用途 |
|------|----------|------|
| `info` | INFO | 一般信息提示 |
| `tip` | TIP | 有用的建议或技巧 |
| `warning` | WARNING | 需要注意的警告 |
| `danger` | DANGER | 危险操作警告 |
| `details` | DETAILS | 可折叠的详情内容 |

### 示例效果

::: info
这是一条信息提示，用于展示一般性的说明内容。
:::

::: tip
这是一条有用的提示，适合分享技巧和建议。
:::

::: warning
这是一条警告信息，提醒用户注意潜在问题。
:::

::: danger
这是一条危险警告，用于标记可能导致严重后果的操作。
:::

::: details 点击查看详情
这是可折叠的详情内容。

- 支持列表
- 支持其他 Markdown 语法
- 支持代码块
:::

### 自定义标题

在类型后面添加空格和标题文字即可自定义标题：

```markdown

自定义标题的提示框内容。
:::

::: warning 注意事项
自定义标题的警告框内容。
:::
```

## 分隔线

使用三个或更多的 `-`、`*` 或 `_` 创建分隔线：

---

上面是一条分隔线。

### 带文字的分隔线

使用 `---文字---` 格式创建带文字的分隔线：

---示例分隔线---

这种分隔线适合用于分隔文章的不同版本或章节，比如原文和润色版：

---AI 润色版---

## 总结

以上就是博客支持的所有 Markdown 语法。主要特性包括：

- 外部链接自动在新标签页打开
- 图片自动懒加载，点击可放大查看
- 代码块支持自定义标题和语法高亮
- 代码组支持多语言切换展示
- 标题自动生成锚点链接
- 支持带文字的分隔线
- 支持技术文章和散文两种类型
- 支持 VitePress 风格自定义容器（info/tip/warning/danger/details）
